Understanding the Nmap SYN Scan

June 23, 2020

Disclaimer: Unauthorized port scanning is illegal and should not be done. Please ensure you have WRITTEN PERMISSION beforehand.




Nmap is a free and open-source popular port scanner used to detect ports and services being utilized on a network. It is a useful tool to see avenues for exploitation. By default Nmap will scan utilizing the TCP SYN scan, also known as a stealth scan. This method takes advantage of the TCP 3 way handshake process. As a refresher the handshake process operates in this order:

  1. My machine sends a SYN flag to the target machine.
  2. Target machine responds to my machine with a SYN/ACK flag
  3. My machine responds with an ACK flag and thus the connection between the two machines is established.




As part of the stealth scan Nmap will initiate the process to the target host specified to see which port is open. This can be initiated with root privileges running the command: nmap <target host IP> or nmap -sS <target host IP>. If the host responds with a SYN/ACK flag Nmap will list that port as open. Instead of responding with an ACK flag, Nmap will send back a TCP RST flag to kill the connection. An attacker will be able to stealthily perform reconaissance on a target as a firewall would not log an incomplete connection. In short a SYN scan on an open looks like this:

  1. My machine sends a SYN flag to the target machine.
  2. Target machine responds to my machine with a SYN/ACK flag
  3. Port is found open, my machine responds with a RST flag to kill the connection.



This might not often be the case however. If Nmap does not recieve a SYN/ACK flag from the target machine but instead an RST flag this will indicate the port is closed. Nmap may also recieve nothing at all after sending the SYN flag which can be due to usually a firewall, the host being down, or the machine just responding too slowly.
